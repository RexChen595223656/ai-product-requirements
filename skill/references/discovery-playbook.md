# Discovery Playbook / 信息澄清手册

## Purpose / 用途

Use this file only after a product deliverable has already been requested and one narrow information gap still blocks a useful output.
只有在用户已经明确要产出某种产品文档，但还有一两个关键缺口时，才用这个文件补洞。

## Look for

- User role / 用户角色
- Core scenario / 核心场景
- Current workflow / 当前做法
- Pain and consequence / 痛点与后果
- Business goal / 业务目标
- Evidence available / 当前证据
- Constraints / 约束

## Ask for the smallest missing piece

- Prefer one high-value clarifying question.
- If evidence is weak, state hypotheses explicitly.
- End with the next smallest action.
- Do not turn this into a full brainstorming workflow.
- If the whole task is still wide open, hand back to the process skill that is leading ideation.

## Output shape

1. Maturity judgment / 成熟度判断
2. What is known / 已知信息
3. What is assumed / 假设
4. What is missing / 缺失信息
5. Next question or next action / 下一问题或下一动作
