# Requirement Analysis Mode / 需求分析模式

## Trigger

- Interviews, tickets, support feedback, user complaints, competitor notes
- User asks to analyze a demand or problem

## Output structure

For Chinese requests, prefer this structure:

## 需求分析
## 1. 需求概述
## 2. 目标用户
## 3. 核心场景
## 4. 当前做法
## 5. 核心痛点
## 6. 产品机会判断
## 7. AI/Agent 适配性判断
## 8. 能力需求拆解
## 9. 风险与约束
## 10. 当前假设
## 11. 待确认问题
## 12. 下一步建议

For English requests, prefer this structure:

## Requirement Analysis
## 1. Summary
## 2. Target Users
## 3. Core Scenarios
## 4. Current Workflow
## 5. Core Pain Points
## 6. Product Opportunity
## 7. AI or Agent Fit
## 8. Capability Breakdown
## 9. Risks and Constraints
## 10. Current Assumptions
## 11. Open Questions
## 12. Next Step

## Checks

- Do not output only feature suggestions.
- Tie each recommendation to evidence.
- Do not use bilingual headings in normal Chinese analysis output.
